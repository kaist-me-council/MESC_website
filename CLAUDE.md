# Claude를 위한 협업 지시사항

## 1. 개요

이 문서는 Claude가 기계공학과 학생회 웹사이트 프로젝트에 참여할 때, 기존의 디자인 시스템과 UI/UX 가이드라인을 효과적으로 준수하고 협업할 수 있도록 돕기 위해 작성되었습니다.

## 2. 디자인 시스템 준수

모든 UI/UX 관련 작업 시, 프로젝트 루트에 있는 `DESIGN_SYSTEM.md` 파일을 반드시 참조해야 합니다. 이 파일에는 다음 내용이 포함되어 있습니다.

*   **디자인 철학**: 웹사이트의 핵심 디자인 원칙 (전문성, 기술적 미학, 사용자 중심)
*   **컬러 팔레트**: 웹사이트의 주 색상 및 보조 색상 정의 (`globals.css`의 Tailwind CSS 커스텀 속성 활용)
*   **타이포그래피**: 폰트 사용 규칙, 크기, 굵기, 줄 간격, 자간 등
*   **컴포넌트 가이드라인**: Navbar, Footer, Card, Button, Badge 등 주요 컴포넌트의 스타일 및 인터랙션 규칙
*   **유틸리티 클래스**: `globals.css`에 정의된 커스텀 Tailwind 유틸리티 클래스 사용법
*   **애니메이션 및 인터랙션**: 요소별 애니메이션 및 호버 효과 규칙
*   **이미지 및 아이콘**: `lucide-react` 아이콘 사용 원칙 및 이미지 활용 가이드

**핵심**: 새로운 UI 요소를 추가하거나 기존 요소를 수정할 때는 `DESIGN_SYSTEM.md`에 명시된 규칙을 최우선으로 고려해야 합니다. 특히 `globals.css`에 정의된 `--primary`, `--accent` 등의 변수와 커스텀 유틸리티 클래스(`glass-premium`, `tech-mesh`, `hover-lift-premium` 등)를 적극적으로 활용하여 일관성을 유지해야 합니다.

## 3. 코드 스타일 및 구조

*   **Tailwind CSS**: 모든 스타일링은 Tailwind CSS 유틸리티 클래스를 사용하여 인라인으로 적용하는 것을 권장합니다. 복잡한 스타일은 `globals.css`에 커스텀 유틸리티로 추가 후 사용합니다.
*   **Shadcn/UI**: `shadcn/ui` 컴포넌트를 기반으로 개발되었으므로, 해당 컴포넌트의 사용법과 커스터마이징 방식을 따릅니다.
*   **컴포넌트 재사용**: `components` 폴더 내의 기존 컴포넌트를 최대한 재사용하고, 새로운 컴포넌트가 필요한 경우 재사용성을 고려하여 설계합니다.
*   **Next.js App Router**: Next.js App Router 구조를 따르며, 서버 컴포넌트와 클라이언트 컴포넌트의 사용 규칙을 준수합니다.

## 4. 협업 시 유의사항

*   **변경 사항 기록**: 중요한 디자인 또는 기능 변경 사항이 있을 경우, `AGENTS.md` 파일에 변경 내역을 간략하게 기록해 주세요.
*   **질문 및 논의**: 디자인 시스템에 대한 의문점이나 새로운 디자인 제안이 있을 경우, 충분히 논의하여 결정합니다.
*   **테스트**: 변경 사항 적용 후에는 반드시 다양한 해상도와 브라우저에서 UI/UX가 올바르게 작동하는지 확인합니다.

이 가이드라인을 통해 Claude가 프로젝트에 원활하게 기여하고, 웹사이트의 고품질 디자인을 함께 유지할 수 있기를 기대합니다.

## 5. 에이전트 사용 규칙

*   **서브 에이전트 활용**: 코드베이스 탐색, 계획 수립, 구현 등 복잡한 작업 수행 시 **반드시 3개의 서브 에이전트를 병렬로 활용**합니다. 단일 에이전트보다 더 넓은 탐색 범위와 빠른 처리를 위해 이 규칙을 준수합니다.
